# LFS 构建系统

## 项目概述

本项目是一个基于 LFS-BOOK-SYSD-12.4 文档的 Linux From Scratch 构建系统，采用模块化脚本设计，确保构建过程的可重复性和可回溯性。

## 硬件要求

- CPU: 至少4核心（推荐AMD Ryzen 7 9700X）
- 内存: 至少8GB（推荐16GB）
- 硬盘: 至少10GB（推荐30GB，本项目使用/dev/nvme1n1p5，容量29GB）
- 架构: x86_64

## 软件要求

- 宿主系统: Ubuntu 24.04.4 LTS
- 必要工具: bash, gcc, g++, make, binutils, wget, md5sum等

## 执行方法

### 1. 准备工作

1. 确保宿主系统是 Ubuntu 24.04.4 LTS
2. 确保目标硬盘 /dev/nvme1n1p5 已准备好
3. 克隆本项目到 /mnt/lfs 目录

### 2. 执行构建

```bash
# 进入项目根目录
cd /mnt/lfs/LFS

# 执行主控制脚本
bash lfs.sh
```

### 3. 构建过程

构建过程会按照以下步骤执行：

1. **检查宿主系统**：验证硬件和软件要求
2. **分区管理**：格式化目标分区并挂载
3. **软件包下载**：自动下载并校验所有软件包
4. **章节构建**：按文档章节顺序构建系统
   - 章节2：准备宿主系统
   - 章节3：软件包下载与校验
   - 章节4：最终准备工作
   - 章节5：构建交叉工具链和临时工具
   - 章节6：构建临时系统
   - 章节7：进入chroot环境
   - 章节8：安装基本系统软件
   - 章节9：系统配置

## 可回溯性

- 每个主要步骤会生成状态标记文件
- 支持从失败步骤重新开始构建
- 详细的构建日志记录

## 注意事项

1. 构建过程需要较长时间，请确保系统稳定运行
2. 构建过程中需要sudo权限，请确保当前用户有sudo权限
3. 构建过程会占用大量磁盘空间，请确保有足够的临时空间
4. 如遇构建失败，请查看logs目录下的详细日志

## 故障排除

- **分区失败**：检查目标分区是否存在且未挂载
- **下载失败**：检查网络连接和下载链接
- **编译失败**：查看详细日志，检查依赖关系
- **引导失败**：检查GRUB配置和分区设置
