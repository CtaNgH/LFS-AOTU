# Linux From Scratch (LFS) 构建系统

## 项目概述

本项目是一个基于 LFS-BOOK-SYSD-12.4 文档的 Linux 系统构建框架，使用模块化脚本设计，实现了从宿主系统检查到最终系统构建的完整流程。

## 目录结构

```
LFS/
├── lfs.sh              # 主控制脚本
├── packages/           # 软件包相关文件
│   └── packages.csv    # 软件包信息表
├── scripts/            # 构建脚本目录
│   ├── chapters/       # 章节构建脚本
│   │   ├── chapter2/   # 章节 2 构建脚本
│   │   ├── chapter3/   # 章节 3 构建脚本
│   │   ├── chapter4/   # 章节 4 构建脚本
│   │   ├── chapter5/   # 章节 5 构建脚本（交叉工具链）
│   │   ├── chapter6/   # 章节 6 构建脚本（临时工具）
│   │   ├── chapter7/   # 章节 7 构建脚本（chroot 环境）
│   │   ├── chapter8/   # 章节 8 构建脚本
│   │   └── chapter9/   # 章节 9 构建脚本
│   ├── download/       # 软件包下载脚本
│   └── partition/      # 分区管理脚本
├── logs/               # 构建日志目录（自动创建）
├── status/             # 构建状态标记目录（自动创建）
└── README.md           # 项目说明文件
```

## 系统要求

### 宿主系统要求
- Ubuntu 24.04.4 LTS 或兼容系统
- x86_64 架构
- 至少 16GB RAM
- 至少 30GB 可用磁盘空间
- 必要的构建工具：gcc, g++, make, patch, tar, xz 等

### 目标系统要求
- 分区：/dev/nvme1n1p5（可在 lfs.sh 中修改）
- 挂载点：/mnt/lfs（可在 lfs.sh 中修改）

## 使用方法

### 1. 检查宿主系统
```bash
bash lfs.sh --host-check
```

### 2. 分区管理
```bash
bash lfs.sh --partition
```

### 3. 软件包下载
```bash
bash lfs.sh --download
```

### 4. 执行完整构建
```bash
bash lfs.sh
```

### 5. 执行特定章节构建
```bash
bash lfs.sh --chapter 6  # 仅执行章节 6 构建
```

### 6. 清理构建状态
```bash
bash lfs.sh --clean
```

## 脚本功能说明

### 主控制脚本 (lfs.sh)
- 提供命令行界面，支持多种构建模式
- 管理构建状态，支持增量构建
- 调用各个章节的构建脚本
- 提供详细的日志输出

### 章节构建脚本
- 每个章节都有独立的构建脚本
- 负责本章节所有软件包的构建
- 支持跳过已构建完成的软件包

### 软件包构建脚本
- 每个软件包都有独立的构建脚本
- 严格按照 LFS 文档要求实现构建步骤
- 包含错误处理和日志记录
- 构建完成后创建状态标记

## 注意事项

1. **权限要求**：部分操作需要 root 权限，请确保以适当的用户身份执行
2. **环境变量**：确保 LFS 环境变量正确设置
3. **磁盘空间**：确保目标分区有足够的磁盘空间
4. **网络连接**：软件包下载需要网络连接
5. **构建时间**：完整构建过程可能需要数小时

## 故障排除

### 常见问题

1. **分区挂载失败**
   - 检查目标分区是否存在
   - 检查分区是否已格式化
   - 检查挂载点权限

2. **软件包下载失败**
   - 检查网络连接
   - 检查 packages.csv 中的 URL 是否正确
   - 尝试手动下载软件包

3. **编译失败**
   - 检查宿主系统的构建工具版本
   - 检查软件包完整性
   - 查看详细的构建日志

4. **构建状态混乱**
   - 使用 `--clean` 选项清理构建状态
   - 重新开始构建过程

### 日志查看

构建日志存储在 `logs/` 目录中，按时间戳命名。遇到问题时，请查看相关日志文件获取详细信息。

## 安全考虑

1. **以非 root 用户构建**：章节 5 和 6 的构建过程应使用 lfs 用户执行
2. **环境变量检查**：确保 LFS 和 LFS_TGT 环境变量正确设置
3. **路径检查**：确保构建过程中使用的路径正确
4. **权限管理**：确保文件和目录权限设置正确
